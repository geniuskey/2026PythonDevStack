## 🎯 PART 2: 2026 현대 파이썬 표준 스택

**가장 먼저 읽어야 할 핵심 파트입니다.**

### 이 파트가 다루는 것

2026년 기준, 파이썬 실무 개발의 **중심축**이 되는 도구들:

1. **프로젝트 관리 & 패키징** (`uv`, `pyproject.toml`)
2. **코드 품질** (`ruff`, `mypy`)
3. **테스트** (`pytest`, `hypothesis`)
4. **웹 API** (`FastAPI`, `pydantic`)
5. **비동기 I/O** (`async`, `httpx`)
6. **관측성** (`structlog`, `OpenTelemetry`)
7. **데이터 처리** (`polars`, `duckdb`)
8. **LLM 앱** (`langchain`, `vector DB`)

### 왜 중요한가?

이 파트를 읽으면:

- ✅ **"지금 뭘 배워야 하지?"**에 대한 답을 얻습니다
- ✅ **현재 생태계의 흐름**을 파악합니다
- ✅ **취업/이직 시 요구되는 스택**을 이해합니다

### 언제 읽나요?

- **초보자**: 처음부터 순서대로 (전체 그림 파악)
- **경험자**: 관심 분야만 선택적으로 (트렌드 확인)
- **기존 스택 검토**: 사용 중인 도구와 비교

**⏱️ 예상 시간**: 각 챕터 10-15분, 전체 약 2시간

## 🔧 PART 3: 문제 해결 가이드

**실무 시나리오별 즉시 적용 가능한 가이드입니다.**

### 이 파트가 다루는 것

| 가이드 | 내용 | 핵심 라이브러리 |
|--------|------|----------------|
| [API 서버 만들기](../03-problem-solving-guides/api-server.md) | REST API 구축 전체 흐름 | FastAPI, pydantic, httpx |
| [패키지 배포하기](../03-problem-solving-guides/package-deployment.md) | PyPI에 패키지 출시 | pyproject.toml, build, twine |
| [LLM 서비스 구축](../03-problem-solving-guides/llm-service.md) | AI 애플리케이션 개발 | langchain, vector DB, tracing |
| [데이터 파이프라인](../03-problem-solving-guides/data-pipeline.md) | ETL/ELT 구축 | polars, duckdb, dagster |
| [자동화 시스템](../03-problem-solving-guides/automation.md) | 반복 작업 자동화 | schedule, selenium, fabric |

### 이 파트의 특징

각 가이드는:

1. **문제 정의**: "무엇을 만들 것인가"
2. **기술 스택 선정**: "왜 이 조합인가"
3. **구현 흐름**: "어떤 순서로 개발하는가"
4. **실전 팁**: "주의할 점은 무엇인가"

을 포함합니다.

### 언제 읽나요?

- **프로젝트 시작 전**: 기술 스택 선정 단계
- **막혔을 때**: 구현 중 방향성이 필요할 때
- **학습 목적**: 실전 패턴 익히기

**⏱️ 예상 시간**: 각 가이드 20-30분

## 📚 PART 4: 라이브러리 카탈로그

**가장 방대한 파트이자 레퍼런스 섹션입니다.**

### 이 파트가 다루는 것

9개 카테고리, 수백 개 라이브러리의 상세 정보:

- **웹 개발**: 프레임워크, HTTP 클라이언트, 템플릿 엔진
- **데이터 과학**: 분석, 시각화, 수치 계산
- **머신러닝 & AI**: 딥러닝, NLP, 컴퓨터 비전, LLM
- **시스템 & 자동화**: CLI, 스케줄링, 배포, 모니터링
- **문서 처리**: PDF, Excel, Word, 텍스트 파싱
- **멀티미디어**: 이미지, 오디오, 비디오 처리
- **네트워킹**: 소켓, 프로토콜, 크롤링
- **보안 & 암호화**: 인증, 해싱, 토큰 관리
- **유틸리티**: 날짜/시간, 로깅, 설정 관리

### 각 라이브러리 항목 구성

```markdown
## 라이브러리 이름 ⭐

**한 줄 설명**

### 기본 정보
- PyPI: [링크]
- 문서: [링크]
- GitHub: [링크]
- 라이선스: MIT

### 언제 사용하나요?
구체적인 사용 사례

### 설치 및 기본 사용
간단한 예제 코드

### 2026 평가
- 장점
- 단점
- 대안

### 함께 사용하면 좋은 라이브러리
관련 도구들
```

### 언제 읽나요?

- **탐색 모드**: "이런 것도 파이썬으로 할 수 있나?" 궁금할 때
- **선택 고민**: 여러 옵션 중 비교가 필요할 때
- **레퍼런스**: 특정 라이브러리의 빠른 정보 확인

**⏱️ 예상 시간**: 필요한 항목만 선택적으로

## 이 책이 대체하지 못하는 것

### 공식 문서

각 라이브러리의 **상세한 API 레퍼런스**는 공식 문서가 훨씬 낫습니다. 이 책은:

- **무엇을 선택할지** (공식 문서는 알려주지 않음)
- **왜 그것인지** (공식 문서는 다루지 않음)
- **어떻게 조합하는지** (공식 문서는 범위 밖)

를 다룹니다.

### 튜토리얼

**단계별 학습**은 전문 튜토리얼이나 강의가 더 적합합니다. 이 책은:

- **방향을 제시**하고
- **선택지를 비교**하며
- **빠른 시작점**을 제공합니다

### 실시간 정보

**최신 버전 변경사항**이나 **실시간 이슈**는 GitHub, PyPI, 공식 블로그를 확인하세요.
